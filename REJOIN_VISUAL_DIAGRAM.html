🎮 Rejoin Guard Architecture

Crash-Recovery System for Halo MCC

❌ WITHOUT OUR FIXES
1 Player in Match
MCC connected to CascadeMatchmaking session
↓
2 MCC Exits
Game closes (crash, quit, kill)
↓
3 Leave Request Sent
PUT {"members":{"me":null}} succeeds → 204
↓
4 Player Removed
MPSD deletes session membership
↓
5 Restart MCC
No memory of session, no session to find
↓
6 ❌ NO REJOIN
Normal matchmaking only
✅ WITH OUR FIXES
1 Player in Match
MCC connected to CascadeMatchmaking session
↓
2 MCC Exits
FIX #2: Watcher detects exit, auto-arms block
↓
3 Leave Request Blocked
FIX #1: Proxy intercepts, returns fake 200
↓
4 Player Stays in Session
MPSD never receives leave → session intact
↓
5 Restart MCC + Restore
FIX #3-6: Reconstruct session from saved state
↓
6 ✅ REJOIN PROMPT
Queue again → MPSD recognizes session → rejoin

The 7 Fixes

Why All 7 Fixes Are Required

  • Without #1: Player removed on first exit → no session to rejoin (404)
  • Without #2: Only works if user manually clicks button → crashes fail silently
  • Without #3/#4: Restarted MCC has no target session → can't attempt rejoin
  • Without #5: Rejoin PUT rejected as unauthorized → 403 error
  • Without #6: MCC thinks no session exists → skips rejoin logic
  • Without #7: ETag mismatch → 412 error loop

Bottom Line: This isn't a native MPSD feature—it's a complete crash-recovery proxy system we had to build because MCC's architecture broke the feature that MPSD was designed to support.
